三个输入全空会直接在第一个地方跳转

输入了用户名之后没有输入序列号，在第四个跳转那边变成了True

三个输入条都有内容之后运行

会在那个jnz跳转，但这次提示的内容不太一样

注册机

要写注册机主要还是要看懂代码

首先用查壳软件看一下编写的语言

那就用DeDeDark看一下对应的代码

发现我们上面修改的关键跳转就是位于button的事件响应函数中

而且Delphi的反编译功能也不是很好，就直接在OD里看吧

看我们修改的那个跳转后面的具体条件

第一处看到上面是cmp dword ptr ds:[0x42f714],0x0这个值不为0就继续执行

我们一共修改了两处关键跳转，正好可以对应两次序列号的比较，推测每次关键跳转上面可能就是对比一个序列号是否正确；

下面我们改动的第一处条件跳转上面没有看到条件

反而是一个call函数，地址0x004838D0

再往上看一些，这附近一共有三个挨着的call函数

正好可以对应一个计算真码、一个计算假码、最后一个比较

单步运行我们可以发现第三个是比较的call，其中传递的参数edx就是我们放着的假码

在0x42d48f处的mov指令将真码放到eax中，看到堆栈可以发现除了我们自己输入的之外多了一串数字

运行到第三个call时查看寄存器发现正好是一个我们自己输入的和一个第一个call计算出来的

之后在第一个关键跳jnz处为真，我们重新运行一次，这次输入的serial1为“1270734”

看看这里是否还会跳转

这一次运行过来时堆栈中的四个值

并且在关键跳转处写着跳转未实现

那么下面的三个call就是同理比较serial2了

如果想写注册机，那么就要具体分析这两个计算真码的关键函数了

Serial1

第一个序列号的真码计算函数很简单

只有这么一小段，但是里面嵌套了一个call

在运行到这个call的时候，eax指向的一片空间存储着name和serial等

跟进去之后计算的真码是保存在[local.1002]中的

实际上又是一个子函数在计算（禁止套娃）

看了半天没什么发现，就去看了看WP，WP有个大佬说序列号的值不是这里算出来的，原本就是有的

跟到42f714看到里面有一个值

这边的两个值转换成十进制就是序列号，这个值一开始就存在了

查找这个地址的参考，发现有一个跳转表

事件函数中还有一个Edit1KeyPress

这个看来是边输入边触发的

每次输入name的一个字符都会运行一遍

之后根据ds:[edx*4+0x42CEC0]的值跳到对应分支，向ds:[0x42f714]和ds:[0x42f718]两个地方加上特定的值

所以注册机就需要写一个大的跳转表，这有点麻烦了，主要是这么多情况抄数据比较慢

Python直接用列表应该就可以

VPS

很早以前就一直想整理这么一个东西，整理一下VPS都能做什么

今天在尝试装一个软件的时候VPS出了一些问题，所以重新装了一些常用的软件，也顺便整理一下吧

Blog

写博客可以说是最基本的功能之一了

在github的gitpage中可以直接用的几个常用静态网页框架都可以直接在VPS上部署博客

复杂一些的还可以用Wordpress，像我就为了方便直接也是用了hexo

一般在重装VPS之后要部署博客都做这么几件事：

  1. 修改ssh端口
  2. 在root的目录下心间一个dir
  3. 将这个dir设置为bare的git repository
  4. 将博客clone在网站代理的目录

具体需要

这个文件内容

由于我的博客是一式两份部署在了github page和VPS上，所以直接可以这样做

这样每次更新博客，这个repo收到deploy的要求之后就会自动执行git hooks去相应的目录更新网站内容

Anki Server

Anki是一个记忆的软件，可以规划好时间背单词，会自动按照记忆曲线分配每天需要背的任务，Anki也有非常强的功能，可以用CSS、html等内容定制模板

并且是一个全平台的软件，所以一个比较好的学习方式就是在电脑上学习时制作卡片，然后每天用手机来学习；

但是这个软件有个问题就是服务器在美国，而且由于人家是非盈利的，就导致国内速度特别慢；

而有一个VPS就可以在自己的VPS上部署服务器，这样同步速度完全就不是一个档次了

安装方式

官方的说明中说使用nginx还需要特别的配置一下转发

这里就略去了

Proxy

作代理也是一个很重要的功能，国内访问一些网站速度经常比较慢，用VPS访问会快很多，VPS上可以部署ss和v2ray之类的工具

这些就不说怎么安装了，一键脚本多的是

最近我还发现了一个有趣的工具，bbr

它甚至直接给系统换了一个内核，使得网络质量比较差的情况下网速有很大的提升；

直接运行这个指令，然后按照提示安装就可以了，实测确实还是有些效果的

Web Server

Web Server主要可以选择apache或者nginx

网上相关的配置资料也有很多；一般我选择的小型便宜VPS就直接用nginx了，相对可以快一些

安装十分简单

有时候也需要修改一些配置，一般改一下访问权限什么的

最多也就改一改site-available

Frp

用frp可以将内网的机器某个端口映射到服务器的具体某个端口处

不过如果把Windows的3389之类的映射到公网IP，那会有很大概率被攻击

所以这种情况需要设置一个强密码

另外之前也了解到了一些辅助的安全手段，比如port knocking

就是在请求之前必须先发一个特定的包，之后才会进入允许握手的状态

相关资料：

  • https://en.wikipedia.org/wiki/Port_knocking
  • https://wiki.archlinux.org/index.php/Port_knocking

这种手段也可以用在SSH上
